# Laravel 13 Compatibility Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `masterix21/laravel-bookings` compatible with Laravel 13 while keeping Laravel 12 support, verified by the existing Pest suite running green on both versions.

**Architecture:** Configuration/build change plus verification. Widen the `illuminate/contracts` and `orchestra/testbench` constraints, extend the CI matrix with a Laravel 13 row, and run the existing test suite against an actual Laravel 13 install to surface and fix any source-level breaking changes.

**Tech Stack:** PHP 8.4, Laravel 12/13, Pest 3, Orchestra Testbench 10/11, PHPStan/Larastan, GitHub Actions.

## Task 1: Widen dependency constraints in composer.json

**Files:**
- Modify: `composer.json`


to:


## Task 2: Verify the suite passes on Laravel 13

This task installs Laravel 13 locally and runs the suite. It does NOT commit `composer.lock` — the lock is a working artifact for verification only.

**Files:**
- No file changes committed in this task (verification only).

- [ ] **Step 1: Install the Laravel 13 dependency set**

Run:

```bash
composer require "laravel/framework:13.*" "orchestra/testbench:11.*" --no-interaction --no-update --dev
composer update --prefer-stable --prefer-dist --no-interaction
```

Expected: resolution succeeds; `composer show -D` lists `laravel/framework 13.x` and `orchestra/testbench 11.x`.

Note: `--no-update --dev` adds `laravel/framework` to `require-dev` temporarily for the local install only. Discard this composer.json change before finishing (Step 5).

- [ ] **Step 2: Run the test suite on Laravel 13**

Run: `vendor/bin/pest`
Expected: all tests PASS.

If failures occur, STOP and proceed to Task 3 before continuing.

- [ ] **Step 3: Run static analysis on Laravel 13**

Run: `composer analyse`
Expected: `[OK] No errors`.

If Larastan reports it does not support Laravel 13, note the required `larastan/larastan` version — it will be bumped in Task 3.

- [ ] **Step 4: Verify prefer-lowest resolves on Laravel 13**

Run:

```bash
composer update --prefer-lowest --prefer-stable --prefer-dist --no-interaction
vendor/bin/pest
```

Expected: resolution succeeds and tests PASS.

If resolution fails because a dependency's lowest allowed version does not support Laravel 13, note which dependency and its minimum Laravel-13-compatible version — it will be bumped in Task 3.

- [ ] **Step 5: Restore composer.json**

Run: `git checkout composer.json`

This removes the temporary `laravel/framework` dev entry added in Step 1. The `^12.0 || ^13.0` and `^10.0.0 || ^11.0` constraints from Task 1 remain (they are already committed).

Execute this task ONLY if Task 2 surfaced failures. If Task 2 passed entirely, mark this task complete and skip to Task 4.

**Files:**
- Modify: `composer.json` (if a dependency minimum needs bumping)
- Modify: source files under `src/` (if a Laravel 13 breaking change is found)

- [ ] **Step 1: Bump dependency minimums if prefer-lowest failed**

If Task 2 Step 4 failed, raise the minimum of the affected dependency in `composer.json` to its lowest Laravel-13-compatible version. Example for `staudenmeir/belongs-to-through`:

```json

```

If Task 2 Step 3 reported Larastan incompatibility, bump in `require-dev`:

- [ ] **Step 3: Re-run the suite on Laravel 13**

Run: `vendor/bin/pest && composer analyse`
Expected: all tests PASS, `[OK] No errors`.

- [ ] **Step 4: Restore composer.json lock state and commit fixes**

```bash

git commit -m "Fix Laravel 13 compatibility issues"
```

Commit only if Step 1 or Step 2 produced changes. If nothing changed, skip the commit.

## Task 5: Extend the CI test matrix

**Files:**
- Modify: `.github/workflows/run-tests.yml:18-28`

- [ ] **Step 1: Add the Laravel 13 matrix row**

In `.github/workflows/run-tests.yml`, change the `matrix` block:

```yaml
    strategy:
      fail-fast: true
      matrix:
        os: [ubuntu-latest, windows-latest]
        php: [8.4]
        laravel: [12.*]
        stability: [prefer-lowest, prefer-stable]
        include:
          - laravel: 12.*
            testbench: 10.*
```

to:

```yaml
    strategy:
      fail-fast: true
      matrix:
        os: [ubuntu-latest, windows-latest]
        php: [8.4]
        laravel: [12.*, 13.*]
        stability: [prefer-lowest, prefer-stable]
        include:
          - laravel: 12.*
            testbench: 10.*
          - laravel: 13.*
            testbench: 11.*
```

Run: `php -r 'var_dump(is_array(yaml_parse_file(".github/workflows/run-tests.yml")));'` if the `yaml` extension is available; otherwise visually confirm indentation matches the surrounding block (8-space indent for `matrix` keys).
Expected: `bool(true)` or a clean visual check.
